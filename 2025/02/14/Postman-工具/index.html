接口测试工具Postman

​ 在做接口测试时，Postman相当于一个客户端，用于模拟客户发送的各类Http请求，相比于浏览器，Postman可以直观地以json格式输出返回数据，方便测试。

接口测试流程

1、接口测试分类

​ 在测试外部接口和供内部使用的内部接口时，一般正常调用就行，而如果测试的是供外部使用的内部接口，就需要测试地非常全面。

2、接口测试重点

  • 参数传递的正确性
  • 接口功能的正确性
  • 各种异常情况下系统的容错能力
  • 鉴权
  • 兼容性

3、接口测试流程

  1. 拿到API接口文档，熟悉接口文档业务、接口地址、请求方式、鉴权方式、入参、出参、是否有完善的错误码机制；

  2. 编写接口测试用例；

    正例

    反例：

    • 鉴权反例：必填、错误的鉴权码、鉴权码过期……
    • 参数反例：必填、参数类型、长度……
    • 错误码反例
    • 黑名单、接口调用次数限制、分页等场景
    • 兼容性：是否能兼容多个版本的前端

  3. 使用接口测试工具执行接口测试，记录缺陷；

  4. Postman+Newman+Jenkins+git实现持续集成并生成报告

接口关联&环境变量

​ 有时候测试一个接口需要用到类似token这样的鉴权码，而鉴权码通常来自于另一个生成鉴权码的接口，这样接口之间就产生了关联。此时可以先调用生成鉴权码的接口，得到返回值token，让后把它保存为全局变量，在需要的地方调用即可。

正则表达式

  • .可以匹配除了\n和\r之外的任意单个字符;

  • *匹配0个或多个正好在它之前的那个字符，比如：ab*c可以匹配ac、abc、abbc等等;

  • ?匹配0个或1个正好在它之前的那个字符，比如：ab?c

可以匹配ac、abc；（非贪婪匹配）

  • 合在一起呢？它就表示匹配任意字符到下一个符合条件的字符，比如：a.*bc可以匹配abc、abbc、axbc等等，这里的".*?"就表示匹配双引号里的内容。

拓展一下：+表示匹配一次或多次，相比于*，它不能匹配0次；

断言

断言在Test操作框中设置，用JavaScript编写，结果在TestResult内。Postman内置了常用的八种断言：

  • Status code:Code is 200———————————-检查返回的状态码是否为200
  • Response body:Contains string————————-检查返回的数据中包括有指定的字符串
  • Response body:JSON value check———————检查json中的其中一个字段的值
  • Response body:Is equal to a string———————检查返回的值等于一个指定的字符串
  • Response headers:Content-Type header check—-检查是否包含响应头Content-Type
  • Response time id less than 200ms———————-检查请求耗时小于200ms
  • Status code:Successful POST request—————–检查返回的状态码是否在数组中
  • Status code:Code name has string———————-检查状态信息是指定的字符串

注意事项：

  • 一旦需要断言随机数，那么必须使用自定义动态参数

  • 在断言中不能使用{{}}的方式取全局变量，要通过pm.globals.get("变量名")方式获取全局变量

  • 在断言中取数据文件的值，不能使用pm.globals.get("times")方式，需要使用文件名.文件字段名

参数化

适用场景：当一个接口需要测试多种输入情况时，比如正向用例、反向用例，可以使用数据驱动的形式组织。

可以使用csv文件和json文件来组织数据，然后在批量执行中选择要测试的用例、执行次数、执行间隔时间和数据文件即可。

Cookie鉴权

当客户端第一次访问服务器时，服务器会生成Cookie，并通过响应里的Set-Cookie值传递给客户端，客户端存储在本地，（F12的Application）下次访问时会自动读取本地的Cookie，然后根据主机IP或域名添加对应的Cookie，实现鉴权。

Cookie分为会话Cookie和持久Cookie，我们可以设置会话Cookie的失效时间。

postman会自动填写Cookie，不用手动添加，比较方便。

Mock测试

场景：由于项目中不同分工，会出现每个人的任务进度不一样的情况，比如A模块完成开始测试，但的它依赖模块B还未完成，测试无法推进，这时使用Mock服务模拟发送预期结果就可以进行测试了。

创建Mock Server，编写好预期结果，Postman会自动创建好Collections，直接发送请求即可。

处理加/解密接口

加密技术：

  • 对称式加密：DES、AES、BASE64
  • 非对称式加密（公私钥）：RSA
  • 只加密不解密：MD5、SHA1、SHA3
  • 混合加密（自定义加密方式）：使用多种加密方式
  • 接口签名Sign

Https使用的是对称加密还是非对称加密？

在数据传输阶段使用的是对称加密，而在证书验证阶段使用的是非对称加密。

Newman工具

定位：postman命令行工具

  • 安装Newman：npm install -g newman，安装html 报告模板方便查看测试用例执行结果：npm install -g newman-reporter-html

  • 导出用例、环境变量和全局变量为三个json文件，注意不要用中文名。

  • 使用语法：newman run 用例json文件路径，参数：

参数名
-e – environment 后面跟环境变量文件
-g – global 后面跟全局变量文件
-n – iteration-count 后面跟迭代次数
-d – iteration-data 指定用于迭代的数据源文件
-r – reporters 后面跟数据结果
html 将结果生成为html文件，指定生成到具体目录需要配合 –reporter-html-export 文件名 使用
