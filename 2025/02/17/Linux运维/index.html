这里列出一些常用命令（自己不熟悉）

find

命令：find 目录 参数 文件名称
示例：find /usr/tmp -name 'a*' 查找/usr/tmp目录下的所有以a开头的目录或文件。

VIM

vi编辑器的3种模式：分别是命令模式（command mode）、插入模式（Insert mode）和底行模式（last line mode），各模式的功能区分如下：

  1. 命令行模式

    控制屏幕光标的移动，字符、字或行的删除，查找，移动复制某区段等。

    dd：删除当前行

    /字符：查找字符

    :：进入底行模式

    i\o\a：进入编辑模式

  2. 编辑模式

  3. 底行模式

    将文件保存或退出vi，也可以设置编辑环境，如寻找字符串、列出行号……等。

查看文件

cat：查看文件最后一屏

more：百分比显示，可以显示百分比，回车下一行，空格下一页，按 q 退出。

less：翻页查看，PageUp/PageDown翻页

tail -行数：查看后n行，Ctrl + C结束

权限

使用ll查看文件或目录权限，第一位表示文件类型：

d：文件夹
-：普通文件
l：软链接（类似Windows的快捷方式）
b：块设备文件（例如硬盘、光驱等）
p：管道文件
c：字符设备文件（例如屏幕等串口设备）
s：套接口文件

chmod u+x aaa.txt：为aaa.txt加上当前用户可执行权限（所在组是 g ，其他用户是 o）。

或者八进制421法：chmod 777 aaa.txt

文件压缩与解压

一般是打 tar 包，压缩为 gzip 包。

  1. 压缩

    tar -zcvf：其中，z 表示调用 gzip 压缩命令进行压缩，c 表示打包文件，v 显示运行过程，f 指定文件名。

  2. 解压

​ tar -zxvf 压缩包 -C 解压路径

查找命令

grep

grep命令是一种强大的文本搜索工具，举例：

find

在目录结构中搜索文件，并对搜索结果执行指定的操作。

locate

locate 让使用者可以很快速的搜寻某个路径。默认每天自动更新一次，所以使用locate 命令查不到最新变动过的文件。为了避免这种情况，可以在使用locate之前，先使用updatedb命令，手动更新数据库。

如果是精简版CentOS系统需要安装locate命令：yum -y install mlocate 。

whereis

whereis命令是定位可执行文件、源代码文件、帮助文件在文件系统中的位置。这些文件的属性应属于原始代码，二进制文件，或是帮助文件。

which

which命令的作用是在PATH变量指定的路径中，搜索某个系统命令的位置，并且返回第一个搜索结果。

主机名配置

vi /etc/sysconfig/network

IP地址配置

vi /etc/sysconfig/network-scripts/ifcfg-eth0

域名映射

vi /etc/hosts

windows域名映射文件路径：C:\Windows\System32\drivers\etc\hosts

ifconfig（net-tools套件中停止维护的命令）

示例

ip（iproute2套件中的命令）

示例

ping

ping 命令用于检测主机，执行 ping 指令会使用 ICMP 传输协议，发出要求回应的信息，若远端主机的网络功能没有问题，就会回应该信息，因而得知该主机运作正常。

route

route 命令用于显示和操作 IP 路由表。

lsof

lsof（list open files）是一个查看进程打开的文件的工具

示例：

netstat

netstat 用来查看当前操作系统的网络连接状态、路由表、接口统计等信息，来自于 net-tools 工具包，ss 是 netstat 的升级版。

常用选项

参数 含义
-a 显示主机中所有活动的网络连接信息 (包括监听、非监听状态的服务端口)
-n 以数字的形式显示相关的主机地址、端口等信息
-p 显示与网络连接相关联的进程号、进程名称信息 (该选项需要 root 权限)
-l 显示处于监听 (Listen) 状态的网络连接及端口信息
-t 查看 TCP (Transmission Control Protocol，传输控制协议) 相关的信息
-u 显示 UDP (User Datagram Protocol，用户数据报协议) 协议相关的信息
-r 显示路由表信息
-i 显示网卡列表
-g 显示组播组的关系
-s 显示网络统计信息

ss

ss 命令来自于 iproute 包，是 netstat 的升级版本。netstat 通过遍历 /proc 来获取 socket 信息，ss 使用 netlink 与内核 tcp_diag 模块通信获取 socket 信息。

traceroute

raceroute 命令可以用于测试从当前主机到目的主机之间经过了哪些网络结点，并显示各个中间结点的连接状态（响应时间）。 yum -y install traceroute 安装。

nslookup

域名解析工具，执行 yum -y install bind-utils 安装。

tcpdump

网络数据包截获分析工具。

详细使用

arp

arp协议：是（Address Resolution Protocol）地址解析协议的缩写。它的作用是通过目标设备的ip地址，解析为目标设备的mac地址，以此来保证信息的传输。他是ipv4网络层不可获缺的协议，不过在ipv6中被ndp（邻居发现协议）代替。

arp缓存就是用来储存ip地址和mac地址的缓冲区，本质就是一个IP地址对应mac地址的对应表。

arp 命令用于操作主机的 arp 缓冲区，可以用来显示 arp 缓冲区中的所有条目、删除指定的条目或者添加静态的 ip 地址与 MAC 地址对应关系。

定时任务

一次性计划任务 – at

周期性计划任务 – crontab

crontab是Unix和Linux用于设置定时任务的指令。通过crontab命令，可以在固定间隔时间,执行指定的系统指令或shell脚本。时间间隔的单位可以是分钟、小时、日、月、周及以上的任意组合。yum install crontabs安装。

进程管理

进程概念

Process: 进程是程序在数据集上的一次执行，运行中的程序的一个副本，是被载入内存的一个指令集合，是资源分配的单位。

进程生命周期

线程和协程

线程：有时被称为轻量级进程，是操作系统调度（CPU调度）执行的最小单位。一个进程实际上可以由多个称为线程的执行单元组成，每个线程都运行在进程的上下文中，并共享同样的代码和全局数据。进程的个体间是完全独立的，而线程间是彼此依存的。多进程环境中，任何一个进程的终止，不会影响到其他进程。而多线程环境中，父线程终止，全部子线程被迫终止(没有了资源)。

协程：是用户模式下的轻量级线程，最准确的名字应该叫用户空间线程（User Space Thread）。协程不是被操作系统内核所管理，而是完全由应用程序来控制。一个线程可以包含一个或多个协程，协程拥有自己的寄存器上下文和栈，协程调度切换时，将寄存器上下文和栈保存起来，在切换回来时恢复先前保运的寄存上下文和栈。

进程管理常用命令

ps

（process status）命令用于显示当前进程的状态。

参数 含义
-a 显示当前终端下的所有进程信息，包括其他用户的进程。与“x”选项结合时将显示系统中所有的进程信息
-u 使用以用户为主的格式输出进程信息
-x 显示当前用户在所有终端下的进程信息
-e 显示所有进程
-f 使用完整的格式显示进程信息
-l 使用长格式显示进程信息

示例：

top

用于查看进程动态信息。

参数 含义
d 改变显示的更新速度
c 显示整个命令行而不只是显示命令名
p 通过指定 PID 来监控某个进程的状态
S 累积模式，会将己完成或消失的子行程的 CPU time 累积起来
s 安全模式，将交互式指令取消, 避免潜在的危机
i 不显示任何闲置或者僵死进程
n 更新的次数，完成后将会退出 top

pgrep

通过特定条件查询进程信息。

参数 含义
-l 同时显示进程名和 PID
-P 指定父进程号
-u 匹配有效用户 ID
-U 匹配实际用户 ID
-g 匹配列出的进程组 ID
-G 匹配真实组 ID
-t 指定开启进程的终端
-o 当匹配多个进程时，显示进程号最小的那个
-n 当匹配多个进程时，显示进程号最大的那个

示例：

pstree

以树形结构显示显示进程的父子关系。

参数 选项
-a 选项可以列出完整的命令信息
-p 选项使用时可以同时列出对应的 PID 号
-u 选项可以列出对应的用户名

示例：

prtstat

可以显示进程信息,来自于 psmisc 包

示例：

uptime

系统负载查询

w&who

w 命令与 who 命令可以用来查看当前系统中登录的用户信息，这是通过读取 /var/run/utmp 文件中的信息来实现的。但是并非所有的程序都使用 /var/run/utmp 文件来保存登录信息，因此系统中实际在线的用户数目可能要比 w 命令与 who 命令的输出多。
